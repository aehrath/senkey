SenKey Help

This guide explains how to use SenKey using only the screens and controls visible in the extension popup.

This help page is built into the extension so you can reopen it any time from the popup header.

Initial setup

  1. Click the SenKey icon in your browser toolbar.
  2. Open ⚙.
  3. In Google Account, click Sign In With Google.
  4. Complete the Google sign-in window.
  5. In Server Configuration, enter your API URL.
  6. Enter your API Key.
  7. Click Save Settings.

After this is done, SenKey is ready to fetch, save, and fill your credentials.

Fill tab

The Fill tab lists your saved credentials. Each credential card shows:

  • the website icon
  • the saved domain
  • a ▶ fill button
  • a ✎ edit button
  • a ✕ delete button

Fill a credential

Click the card or the ▶ button. SenKey will fill the current login page or open the saved login page and fill there.

When the current tab is already a saved login page, SenKey briefly shows an Active fill button at the top of the credential list.

Edit a credential

Click ✎. SenKey opens the credential in the Add tab with the current values prefilled.

Delete a credential

Click ✕. If deletion succeeds, SenKey shows Deleted.

Folders

Credentials can be grouped into folders. Folders appear above ungrouped credentials and can be collapsed or expanded — SenKey remembers the open/closed state between sessions. Nested folders use / as a separator (e.g. Work/Clients).

Folder paths are saved with your credentials in the SenKey bucket, so they follow the signed-in Google user across browsers.

Click ✎ on a folder header to rename that folder path. Renaming to an existing folder path automatically merges the credentials into that folder.

Add tab

Use the Add tab to save a new credential or update an existing one.

Visible fields:

  • Domain
  • Username / Email
  • Password — click ↻ to suggest a secure password.
  • Login URL
  • Folder — typing a name groups the credential under that folder in the Fill tab. Use / for nested folders (e.g. Work/Clients). Existing folder names are suggested as you type.

Save a new credential

  1. Open Add.
  2. Enter the site in Domain.
  3. Enter your username or email.
  4. Enter your password, or click ↻ to suggest a secure password.
  5. Optionally keep or change Login URL.
  6. Click Save To Server.

Update an existing credential

  1. Start from Fill.
  2. Click ✎ on a saved credential.
  3. Edit the values you want to change, including Folder if needed.
  4. Click Update Credential.

Password visibility

Click the 👁 button next to the password field to show or hide the password.

Password suggestion

Click the ↻ button next to the password field to fill a new 20-character password suggestion. The suggestion is generated locally in the extension. It is saved and encrypted only when you click Save To Server or Update Credential.

Settings tab

Google Account

If you are signed out, you will see Sign In With Google. If you are signed in, you will see your profile initial, your name, your email, and a ✕ sign-out button.

In Brave, enable brave://settings/extensions > Allow Google login for extensions, then sign into Google in a normal Brave tab before signing into SenKey.

If you are testing an unpacked custom build, run ./build.sh and load dist/. Dev builds show a DEV badge on the extension logo so they are easy to tell apart from the published build. The OAuth client IDs in .env must match that installed extension ID.

If Google shows redirect_uri_mismatch, the Web application OAuth client used for Brave fallback is missing the exact https://<extension-id>.chromiumapp.org/oauth2 redirect URI, or the extension was built with the wrong OAuth client type.

Server Configuration

Enter API URL and API Key, then click Save Settings.

Encryption Keys

Use Export Keys and Import Keys to back up or restore your SenKey encryption keys.

Keep exported keys safe. SenKey uses these keys to decrypt your saved passwords. If they are lost and your browser storage is gone, you may not be able to recover your saved passwords. Anyone with this key file may be able to read the passwords protected by it.

Login Pages

Use Export Login Pages to download a JSON backup of the browser login page list. SenKey also tries to save the same backup to the bucket document that stores your credentials and credential folder paths. If the bucket save fails, the local JSON download still completes and SenKey shows the bucket error.

Use Import Login Pages to restore a login page JSON backup into a new browser folder and replace the bucket login page backup. Existing login pages are not deleted or overwritten.

If login pages change while the popup is open, SenKey saves the updated login page list to the bucket automatically and tries one last save when the popup closes.

For system administrators

SenKey needs a deployed backend before users can fetch, save, or sync credentials. The backend-only deployment zip is available here: senkey-deploy.zip.

Deploy the backend, then give users the deployed API URL and API Key for the extension Settings tab.

For custom extension builds, use a Chrome Extension OAuth client for GOOGLE_OAUTH_CLIENT_ID and a separate Web application OAuth client for WEB_GOOGLE_OAUTH_CLIENT_ID. Do not reuse the Chrome Extension client as the Web client.

If the extension does not work after sign-in and settings are saved, ask your system administrator to deploy or check the SenKey backend.

Typical daily use

  1. Open the site you want to sign in to.
  2. Open SenKey.
  3. In Fill, click the matching credential.
  4. If you need to save a new login, use Add.
  5. If you need to change a password later, use ✎ from Fill.
